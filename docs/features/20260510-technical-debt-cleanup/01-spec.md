# Spec: 技术债清理项目

## 元数据

- **创建日期:** 2026-05-10
- **artifact_type:** software（自动化脚本 + 文档改进）
- **优先级:** P0（合同漂移） + P1（版本同步、历史文档）
- **目标版本:** v2.15.0

## 问题陈述

Unified Skills 项目存在以下技术债：

1. **合同漂移（P0）** - skills-index.json、版本号、文档引用需要手动同步，导致频繁的 drift 修复
2. **历史文档污染（P0）** - 5 个特性文档需要明确的过期标记，避免误导新用户
3. **版本同步负担（P1）** - 34 个文件包含版本号，发版时容易遗漏
4. **验证脚本复杂度（P0）** - 632 行验证脚本，检查项持续增长

## 目标

### 主要目标

1. **自动化版本同步** - 创建脚本自动同步所有版本号，减少人为错误
2. **自动化索引生成** - 创建脚本自动生成 skills-index.json
3. **完善历史文档标记** - 为所有过期文档添加明确的标记
4. **简化验证脚本** - 集成自动化工具，减少手动检查项

### 次要目标

1. 建立技术债监控机制
2. 完善文档结构说明

## 约束

1. **零依赖原则** - 不能引入外部依赖，只能使用 bash/python3/git
2. **向后兼容** - 不能破坏现有的工作流和文档结构
3. **验证优先** - 所有自动化脚本必须包含测试
4. **文档同步** - 修改后必须更新相关文档

## 验收标准

1. [ ] 版本同步脚本能正确更新所有 3 个插件元数据文件
2. [ ] skills-index.json 生成脚本能正确提取所有技能信息
3. [ ] 所有历史文档都有明确的"历史 / 已过期"标记
4. [ ] validate 脚本集成新的自动化检查
5. [ ] 所有脚本都有对应的测试
6. [ ] 文档已更新，说明新的自动化工具使用方法

## 风险

1. **自动化脚本错误** - 可能导致错误的版本号或索引
   - 缓解措施：完善的测试 + 人工验证步骤

2. **文档标记混淆** - 可能误标记活跃文档为历史
   - 缓解措施：人工审核所有标记

3. **验证脚本破坏** - 集成新检查可能破坏现有验证
   - 缓解措施：保留所有现有检查，只添加新检查
