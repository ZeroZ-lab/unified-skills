# Direction Design — 导演设计

## 入口/出口
- **入口**: `deck` 或强呈现导向内容需要定义演示推进和场面调度
- **出口**: 页面切换意图、情绪推进、演讲节奏和呈现编排定稿
- **指向**: 剧本主线 → `design-content-script`；页面落地 → `design-content-layout`
- **输出路径**: `build-content-writing` 或 `build-content-layout`（下游 build 技能）
- **前置加载**: CANON.md + `design-workflow-design`

## 何时不使用
- 普通长文阅读材料
- 只做静态版式、不涉及讲述推进的视觉稿

## 核心原则

1. **Sequence Creates Meaning**
2. **Reveal Is a Design Decision**
3. **Emotion Needs Control**
4. **Speaker Load Matters**
5. **Every Transition Must Earn Its Keep**

## 最佳实践输入

先读取 `references/design-best-practices.md` 和 `references/design-inspiration-catalog.md`，并把导演相关证据写入 `02-design.md` 的 `Design References / Pattern Synthesis / Adopt / Reject`。

扫描重点：
- Enterprise Product Patterns: pitch deck、发布会、产品演示、汇报材料的页序和揭示模式
- Official Systems / Platform Rules: 演示媒介、时长、品牌语气、动画/导出限制
- Methods / Theory / Style Schools: 戏剧张力、揭示顺序、节奏控制、speaker load 管理
- Anti-patterns / Verification: 每页平均用力、无转折、动画代替叙事、一页多功能
- Local Project Truth: 观众任务、现场/异步阅读场景、页数限制、演讲者能力和素材边界；项目根 `DESIGN.md`（如果存在，作为导演约束参考）

导演设计必须说明为什么这样推进；不能把页序写成材料清单。

## 流程

### Step 1：定义演示目标
- 观众在每个阶段应该感受到什么
- 什么时候建立张力
- 什么时候交付结论
- ✅ 验证点: 演示目标是否明确区分了"每个阶段的感受"，而非笼统的"让观众理解"？

### Step 2：规划推进
- 页与页之间如何接力
- 哪些内容应该单独成页
- 哪些内容必须成组出现
- ✅ 验证点: 页间接力是否有明确的叙事意图（如"此处转折制造意外"），而非只是内容排列？

### Step 3：控制节奏
- 快切还是停顿
- 哪里需要解释空间
- 哪里只给结论不展开
- ✅ 验证点: 节奏快慢切换是否有具体目的标注（如"此处快切3页制造紧迫感，随后1页停顿消化"）？

### Step 4：记录呈现约束
- speaker notes 是否承担细节
- 页面是否需要动画/渐进披露
- 现场阅读 vs 投屏演讲
- ✅ 验证点: 呈现约束是否具体到场景（如"投屏演讲，每页停留 30-60 秒"），而非"到时候再说"？

## 输出契约

写入 `02-design.md`：
- 演讲节奏
- 页序 / 段落推进
- 情绪推进 / 转折点
- Adopt / Reject（导演模式）
- 不做清单

## 验证失败处理

| 失败场景 | 处理方式 |
|---------|---------|
| 页序只是在平铺信息 | 重写推进逻辑，增加转折和揭示顺序 |
| 节奏过密 | 拆页或让 speaker notes 承担细节 |
| 节奏过散 | 合并页面，强化主线 |
| 推进模式无证据 | 补充 best-practice scan，写清 Adopt / Reject |
| 情绪推进与故事线冲突 | 回到故事脊柱对齐情绪节奏，删除不服务主线的转折 |

## 常见说辞

| 说辞 | 现实 | 后果 |
|------|------|------|
| “把页做出来就有节奏了” | 没有导演设计的页序只是在排材料。 | 无导演设计 → 页序变成信息平铺，听众在 5-8 页后注意力断线，关键转折点被淹没 |
| “动画以后再说” | 是否渐进披露会影响叙事结构。 | 动画未提前规划 → 结构性揭示需要重排页序，返工 30-50% 的 deck 内容 |
| “每页都讲一点” | 平均分配信息通常等于没有重点。 | 平均用力 → 无重点页序让听众无法识别核心主张，演示说服力下降 40%+ |

## 好坏示例

### ✅ Good: 证据驱动的导演决策

```
演示目标:
- 前 3 页: 让观众感到"这是我们都知道的痛点"（共鸣）
- 第 4-6 页: 制造张力——"直觉解是错的"（转折）
- 第 7-8 页: 交付结论和行动（收束）

推进逻辑:
- 页 1→2: 快切（痛点堆叠制造紧迫感）
- 页 3→4: 停顿（转折前需要消化空间）
- 页 4→5→6: 快切3页（证据密集，不容犹豫）
- 页 6→7: 停顿（结论前让听众准备接受）
- 页 7→8: 单页行动清单（收束干净）

Adopt: 问题-转折-收束三段推进（来源: Methods / Theory / Style Schools #5）
Reject: 匀速信息排列（与演示目标"制造张力再收束"冲突）
```

### ❌ Bad: 抽象模板无具体决策

```
演示目标: 让观众了解产品
推进逻辑: 按功能列表排列
节奏: 正常速度
呈现约束: (未写)
不做清单: (未写)
→ 没有导演意图，没有情绪推进，没有证据支撑
```

## 输出模板

导演设计产出应写入 `02-design.md` 的以下结构：

```markdown

| 阶段 | 期望感受 | 关键动作 |
|------|---------|---------|
| [阶段 1] | [共鸣/紧迫/惊讶...] | [认同痛点/接受转折/决定行动] |

### 页序推进
| 页码 | 内容 | 推进意图 | 节奏 |
|------|------|---------|------|
| [1-3] | [内容概述] | [制造共鸣] | 快切 |
| [4-6] | [内容概述] | [制造转折] | 快切 |
| [7-8] | [内容概述] | [收束行动] | 停顿 |

### 情绪推进 / 转折点
- 共鸣点: [页码和描述]
- 张力点: [页码和描述]
- 转折点: [页码和描述]
- 收束点: [页码和描述]

### 呈现约束
- 场景: [现场投屏 / 异步阅读 / 混合]
- speaker notes 承担: [哪些细节]
- 渐进披露需求: [哪些页面需要动画]

### Adopt / Reject（导演模式）
| 模式 | 来源 | 决定 | 理由 |
|------|------|------|------|
| [模式 A] | [来源层] | Adopt | [具体理由] |
| [模式 B] | [来源层] | Reject | [冲突点] |

### 不做清单
- [不做项 1]: [冲突来源或证据]
- [不做项 2]: [冲突来源或证据]
```

## 红旗 — STOP

- 没有推进逻辑就开始做 deck
- 缺少导演 best-practice scan 或 Adopt / Reject
- 情绪推进和故事线相互打架
- 一页承担多个阶段性功能
- 把导演设计退化成“加不加动画”
